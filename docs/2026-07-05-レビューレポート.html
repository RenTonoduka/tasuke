DATA LEAK

① 非公開プロジェクトのタスクがREST層で閲覧・改変・削除できる (IDOR)

SEC-H1 / FUNC-H1

MCP層は getAccessibleProjectIds で守っているのに、REST個別ルート(tasks/[id]・search・comments など10ルート)は「ワークスペース所属」しか見ていない。同一WSの一般メンバーが非公開案件(評価・給与等)のタスクを閲覧・改変・削除できる。GET /api/tasks/search がタスクIDまで返すのでID推測も不要。

既存の canAccessProject を各ルートに1行追加するだけの外科的修正。
BROKEN FEATURE

② 自動化ルールが主要経路で発火しない

FUNC-H2 / FUNC-H3

executeAutomationRules の呼び出しがPATCH 1箇所のみ。カンバンD&D・一括更新・MCP/CLI経由のステータス変更では発火しない。DUE_DATE_APPROACHING は評価cron自体が存在せず、UIで作れても永遠に動かない。ユーザーは「設定したのに動かない」状態。

move / bulk / MCP handler に配線。DUE_DATE系はcron評価を追加 or UIから外す。
STRUCTURAL DEBT

③ テストゼロ + タイムゾーン処理が3方式併存

TECH-H4 / TECH-H2

テストファイル0件。「今日」の境界がコード内で3方式併存し、うち2つがVercel上でUTC境界のまま。JST朝9時までダッシュボードとLINEリマインダーが別の答えを返す。TZ修正コミットが繰り返される根本原因で、テスト基盤なしに直すとまた回帰する。

先にvitest導入 + date-jst.ts 集約。以降の修正の回帰防止装置にする。
